# AI_assistent
# ИИ-ассистент для ресторана "На Пару"

Интеллектуальный виртуальный администратор, созданный на базе платформы Suvvy.ai с использованием языковой модели Qwen. Ассистент автоматизирует общение с гостями ресторана "На Пару", предоставляет точную информацию о меню и автоматически управляет бронированием столиков через интеграцию с Google Календарем.

## О проекте

Ресторан "На Пару" специализируется на здоровом питании и блюдах, приготовленных на пару. Гостям важно получать детальную информацию о составе блюд, калорийности, наличии аллергенов и способе приготовления.

Данный проект реализует современного ИИ-агента, который:

- Понимает контекст и ведёт естественный диалог с пользователем
- Мгновенно находит ответы в базе знаний ресторана
- Самостоятельно обрабатывает запросы на бронирование и синхронизирует данные с календарём заведения

Решение построено на связке языковой модели Qwen (для генерации текста и понимания контекста) и платформы Suvvy.ai (для оркестрации агента, управления диалогами и настройки интеграций).

## Функционал

### Осмысленное ведение диалога

Благодаря возможностям модели Qwen, ассистент поддерживает полноценный диалог с пользователем:

- Распознаёт намерения пользователя даже при неформальном стиле общения или опечатках
- Поддерживает контекст в рамках одной сессии диалога
- Адаптирует стиль общения под тональность бренда ресторана: вежливый, информативный, с акцентом на заботу о здоровье
- Обрабатывает многошаговые сценарии: уточнение деталей, подтверждение действий, обработка возражений

### Ответы на вопросы по базе знаний

Система использует технологию RAG (Retrieval-Augmented Generation) для работы с внутренней документацией ресторана:

- Меню и технологические карты: ответы на вопросы о составе, калорийности, наличии аллергенов
- Информация о ресторане: режим работы, адрес, контакты, условия проведения мероприятий
- Концепция и философия: рассказ о технологии паровой обработки, источниках ингредиентов
- Простое обновление знаний: достаточно добавить или изменить документ в базе, без переписывания кода

### Бронирование столиков и интеграция с Google Календарём

Полная автоматизация процесса записи гостей:

- Сбор необходимых параметров бронирования в ходе диалога: дата, время, количество персон, контактные данные
- Проверка доступности столиков согласно настройкам ресторана
- Автоматическое создание события в Google Календаре через API
- Отправка подтверждения пользователю с деталями бронирования
- Возможность изменения или отмены брони через диалог

## Технологический стек

| Компонент | Инструмент | Назначение |
|-----------|------------|------------|
| Языковая модель | Qwen (Alibaba Cloud) | Генерация ответов, понимание контекста, обработка системного промпта |
| Платформа агентов | Suvvy.ai | Создание логики бота, управление диалогами, настройка интеграций и рабочих процессов |
| База знаний | Vector Store (внутри Suvvy) | Хранение и семантический поиск по меню, правилам и справочной информации |
| Календарь | Google Calendar API | Синхронизация бронирований в реальном времени |
| Каналы связи | Telegram, Web Widget | Интерфейсы для взаимодействия с гостями |

## Архитектура решения

```
+----------------+
|    Пользователь |
+--------+-------+
         |
         v
+--------+-------+
|   Suvvy.ai     |
| (Платформа)    |
+--------+-------+
         |
    +----+----+
    |         |
    v         v
+---+---+ +---+---+
| Qwen  | | База  |
| (LLM) | | знаний|
+---+---+ +---+---+
    |         |
    +----+----+
         |
         v
+--------+-------+
| Google Calendar|
|     API        |
+----------------+
```

Поток обработки запроса:

1. Пользователь отправляет сообщение через интерфейс (Telegram или веб-виджет)
2. Платформа Suvvy.ai принимает запрос и определяет интент
3. При необходимости извлечения фактов система обращается к векторной базе знаний
4. Контекст и промпт передаются модели Qwen для генерации ответа
5. Если требуется действие (бронирование), Suvvy вызывает соответствующий инструмент
6. Ответ формируется и отправляется пользователю

## Настройка и развёртывание

Поскольку основная логика реализована на платформе Suvvy.ai, развёртывание не требует сложной инфраструктуры.

### Подготовка базы знаний

1. Подготовьте документы с информацией о меню, составе блюд, ценах, правилах ресторана и часто задаваемых вопросах в форматах TXT, PDF или Markdown
2. Загрузите файлы в раздел Knowledge Base проекта на платформе Suvvy.ai
3. Настройте параметры индексации для оптимального поиска по базе

### Настройка модели и системного промпта

1. В настройках агента выберите модель Qwen (рекомендуется Qwen-72b-chat или Qwen-Max)
2. Настройте системный промпт. Пример:

```
Ты — виртуальный администратор ресторана "На Пару". Твоя задача — помогать гостям выбирать блюда, отвечать на вопросы о составе и способе приготовления, а также бронировать столики.

Будь вежлив, точен и лаконичен. Всегда подчёркивай преимущества блюд, приготовленных на пару.

Если пользователь хочет забронировать столик, уточни: дату, время, количество человек и контактные данные. После подтверждения вызови инструмент бронирования для создания события в календаре.

Если вопрос касается аллергенов или диетических ограничений, отвечай с особой внимательностью и при необходимости рекомендуй альтернативные блюда.
```

### Интеграция с Google Календарём

1. В разделе Integrations платформы Suvvy подключите аккаунт Google
2. Предоставьте необходимые разрешения на управление событиями календаря
3. Настройте действие (Action) для создания события с параметрами:
   - summary: "Бронь: {Имя гостя}"
   - start: {дата и время начала}
   - end: {дата и время окончания}
   - description: "Количество персон: {N}, Контакт: {телефон/почта}"
4. Протестируйте сценарий бронирования в режиме отладки

### Публикация

- Получите токен Telegram-бота или код виджета для веб-сайта в панели управления Suvvy
- Разместите виджет на сайте ресторана или подключите бота к каналу связи

## Примеры диалогов

### Сценарий: Вопрос о составе блюда

Пользователь: Здравствуйте, есть ли в вашем салате "Лёгкий" орехи или глютен?

Ассистент: Добрый день! В салате "Лёгкий" используются свежие огурцы, листья салата, оливковое масло и лимонный сок. Орехи и глютен в этом блюде отсутствуют. Оно полностью подходит для безглютеновой диеты. Могу ли я помочь вам с выбором других блюд или оформить бронирование?

### Сценарий: Бронирование столика

Пользователь: Хочу забронировать столик на завтра в 19:00, нас будет трое.

Ассистент: Конечно. Уточните, пожалуйста, ваше имя и контактный телефон для подтверждения брони.

Пользователь: Анна, +7 (999) 123-45-67.

Ассистент: Спасибо, Анна. Бронь на троих гостей на завтра в 19:00 подтверждена. Я добавил событие в календарь ресторана. Ожидаем вас в ресторане "На Пару". При необходимости вы можете изменить или отменить бронь, написав нам.

## Тестирование

Рекомендуемые сценарии для проверки работоспособности:

1. Проверка понимания контекста: задать серию связанных вопросов и убедиться, что ассистент сохраняет нить диалога
2. Проверка базы знаний: запросить информацию о составе блюд, калорийности, наличии аллергенов
3. Проверка бронирования: пройти полный цикл от запроса до подтверждения и создания события в календаре
4. Проверка обработки ошибок: ввести некорректные данные и убедиться в адекватной реакции системы
